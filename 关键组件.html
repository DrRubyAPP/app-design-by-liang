关键组件库（单独打磨）

复用度高、交互复杂、需沉淀进设计系统的 6 个核心控件。每个组件全状态平铺 + 规格标注（用途 / 出现位置 / 关键 Token / 无障碍），延续 §1 设计 Token 与 §3 界面视觉语言。

基线：app-prompts v1.0 §4
配套 §3 逐屏界面（24 屏）
品牌色 Ruby #C2185B
Rose 主操作/品牌 正 / High / keep 改善 Med / review 中等 负 / 刺激 警示 Low / stop 失败 caution 留意 Zero / skipped 中性

🧩 设计原则：所有语义状态「颜色 + 图标 + 文字」三重编码，不单靠颜色区分（WCAG AA · 无障碍）。每个组件给出可直接落入设计系统的状态集与复用位置（对应 §3 屏号）。

1

AR 采集引导框

AR face-capture guidance overlay · 实时质检 HUD
最高优先级

全屏深色相机之上的引导浮层：椭圆人脸框 + 颧骨/眼下三角靶区 + 距离引导 + 左右侧强制顺序指示 + 4 路实时质检 chip。底层 iOS ARKit / Android ARCore，Flutter 封装为统一组件，UI 两端一致。

漏采降级状态条

Capture-status banner · 3-state

本周采集完成度 3 态：computed（2 次，正常）/ degraded（1 次，置信度下降）/ skipped（0 次，跳过）。语气支持、不施压。

三态状态条
本周进度正常
已完成 2 次采集，趋势可正常解读。
2 / 2
置信度下降 — 本周仅 1 次
周末前请再采集一次，以恢复本周趋势置信度。
1 / 2
本周已跳过
没关系，下周继续即可，不影响整体研究。
0 / 2
用途
告知本周采集完成度对趋势置信度的影响，缺采时温和召回而非批评。
出现位置
屏 16 上传完成（降级周）、首页采集提醒、SIS 结果置信度说明。
关键 Token
正常 #2E9E6B · 降级 #E0A23A · 跳过 #9AA0A6；N/2 计数 pill。
无障碍
图标 + 文案 + N/2 计数三重表意；语气支持，避免负罪感。
